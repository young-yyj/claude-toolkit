# PRD Writer — 结构化产品需求文档生成器

## 这是什么？

`sk-prd-writer` 是一个严格结构化的 PRD 生成 Skill，通过渐进式对话从三个视角（用户 / 商业 / 技术）诊断需求，按阶段产出从概念版到落地版的完整产品需求文档，专为 **vibe coding** 场景设计。

## 触发方式

直接用自然语言描述你产品想法、功能需求或改进意见即可触发。例如：

- "帮我写一个记账 App 的 PRD"
- "我想做一个小程序，帮用户找附近的咖啡馆"
- "现有后台管理系统需要加一个权限管理模块"

> 无需特定关键词，Skill 会自动识别产品需求场景。

## 流程概览

| 阶段 | 做什么 | 产物 |
|------|--------|------|
| Phase 1 — 三视角诊断 | 用户/商业/技术三视角逐轮提问（≤3 问题/轮） | 需求信息全集 |
| Phase 2 — 概念版 | 输出 ≤200 字概念版 PRD，对齐方向 | 概念版 PRD |
| Phase 3 — 范围冻结 | 锁定目标用户、核心功能、平台、排除项 | 冻结确认 |
| Phase 4 — 落地版 | 按四层结构输出完整 PRD + 六大盲区自检 | 完整落地版 PRD |

## 落地版 PRD 结构

1. **概述** — 需求背景、目标、范围、参考资料
2. **功能列表** — 模块、功能、优先级（P0/P1/P2）
3. **页面结构与导航设计** — 整体导航、每个页面职责与元素、独立页面/弹窗
4. **功能说明** — 每个功能含：用户流程（含异常路径）、状态机、字段规范、文案规范、异常情况处理

## 六大盲区自检

落地版输出前自动检查每个功能是否覆盖：

- 页面结构：导航、页面职责、元素、独立页面/弹窗
- 异常交互：用户流程中的异常分支
- 状态机：完整状态及进入/退出条件
- 字段规范：类型、必填、长度、校验、错误提示
- 文案规范：空状态、加载、成功、失败、权限、网络异常
- 异常情况处理：网络断开、数据为空、操作超时、权限不足

## 约束

- 落地版不允许出现"待定"、"TBD"等模糊表述
- 概念版确认前不输出任何完整 PRD 或功能列表
- 总字数限制：概念版 ≤200 字

## 适合谁用？

- 有产品想法但不知如何结构化的开发者
- 需要用 vibe coding 快速实现产品的团队
- 希望 PRD 覆盖全面（五大盲区）的产品经理
