# 变更→文档映射规则

根据 Step 1 产出的变更文件清单逐文件分析，判断需要更新哪些文档。

## 规则表

| 变更类型 | → CLAUDE.md | → README.md | → README_CN.md | → architecture.md | → files.md | → conventions.md |
|----------|-------------|-------------|-----------------|-------------------|------------|------------------|
| 新增源文件 (`A src/...`) | 如影响架构链则更新 | 如影响项目结构树则更新 | 跳过 | 如引入新模块/通信路径则更新 | **必须更新**—添加文件条目 | 如引入新模式则更新 |
| 删除源文件 (`D src/...`) | 同上 | 同上 | 跳过 | 如移除模块则更新 | **必须更新**—移除文件条目 | 如废弃模式则更新 |
| 重命名文件 (`R...`) | 同上 | 同上 | 跳过 | 更新路径引用 | **必须更新**—更新文件名 | 更新路径引用 |
| 新增 public 方法/类 | 无 | 无 | 无 | 如新增通信路径则更新 | 在关键成员中补充 | 如引入新约定则更新 |
| 新增功能/快捷键 | 如影响构建命令则更新 | **必须更新**—功能列表/快捷键表 | 如影响用户使用说明则更新 | 如引入新交互流则更新 | 无 | 无 |
| 依赖变更 (*.csproj) | **必须更新**—技术栈版本号 | 无 | 无 | 无 | 无 | 无 |
| SKILL.md 变更 | 无 | 无 | **必须更新**—同步触发词/流程/注意事项 | 无 | 无 | 无 |
| 内部实现重构 | 通常跳过 | 跳过 | 跳过 | 通常跳过 | 跳过 | 通常跳过 |

## 各文档分析提示

- **CLAUDE.md**: 重点关注"技术栈"和"架构"行的变更；依赖版本号必须精确
- **README.md**: 面向用户，只更新功能性内容（新功能、新快捷键、新命令）；不要覆盖用户手写的描述性文字
- **README_CN.md**: 面向中文用户的使用说明；当 SKILL.md 的触发词、流程、约束变更时同步更新；README.md 变更时一并检查
- **architecture.md**: 关注通信路径（事件、IPC、WebView2 桥接）、数据流变化
- **files.md**: 最需要精确同步 — 每个源文件的职责描述和关键成员必须反映当前代码
- **conventions.md**: 仅当 diff 中出现了新的命名模式、新的错误处理方式、新的持久化约定时才更新

## 跳过信号

以下情况标记为"建议跳过"：
- 变更文件全部为 `docs/`、`*.md`、`.gitignore` 等非源码文件
- 变更仅修改方法内部实现，无新增 API
