# 教程构建器 (Tutorial Builder)

## 作用

针对指定的 AI 领域知识点，自动进行全网多源资料收集、交叉验证、结构化整理，输出不少于 20000 字的教科书级 Markdown 教程。

## 何时使用

- **生成 AI 领域完整教程**：从入门到精通，覆盖原理、代码、实战
- **系统性地整理 AI 技术方向的知识体系**：如 RAG、RLHF、LoRA 等
- **需要一份高质量学习资料**：含代码示例、来源标注、层次分明
- **用户说 "写一个 XX 教程 / 学习指南 / 从入门到精通"**

不适用于：简短概念解释、仅需大纲/目录、非 AI 领域主题、非 Markdown 格式输出需求。

## 关键功能

### 1. 全网多源资料搜集
- 并行搜索三类来源：官方文档与学术论文（S 级）、中文技术博客（A 级）、微信公众号深度文章
- 对每个来源进行可信度评级（S/A/B/C），高可信优先采用，低可信交叉验证
- 兜底策略：不足 8 个有效来源时放宽时间范围或替换关键词

### 2. 结构化教程生成
- 按"基础篇 → 进阶篇 → 高级篇"分层递进
- 每章开头标注分类标签（`[基础篇]` / `[进阶篇]` / `[高级篇]`）
- 支持四种知识点类型的大纲微调：算法/模型、框架/工具、概念/范式、基础设施/MLOps
- 包含"常见问题与踩坑经验"和"总结与展望"章节

### 3. 内容质量管控
- 总字数 ≥ 20000 字（`wc -m` 精确验证）
- 每个核心概念配有可运行代码示例
- 专业术语首次出现标注英文原文
- 所有引用注明来源 URL，不得出现来源不明的数据或代码
- 教程风格教科书式严谨连贯，杜绝碎片化与炒作式表达

### 4. 微信公众号内容集成
- 强制搜索 `site:mp.weixin.qq.com` 获取中文 AI 领域深度内容
- 限定最近 1-2 年内容，保证时效性
- 优先采用高质量公号（机器之心、量子位、PaperWeekly 等）

## 工作原理

1. **分析知识点**：确定类型、受众层次、前置依赖，提取中英文搜索关键词
2. **并行搜集资料**：同时启动官方文档、中文博客、微信公众号三类搜索，记录搜索词与命中数
3. **内容整合**：多源交叉验证核心概念，按可信度评级筛选，去重整合
4. **选择大纲模板**：根据知识点类型微调通用模板，确定章节结构
5. **顺序生成教程**：从第一章到最后一章顺序撰写，确保前后连贯
6. **输出前自检**：逐条检查 Constraints（字数、日期、来源标注、无占位符）
7. **保存与呈现**：文件名 `{知识点}_从入门到精通.md`，告知用户路径和字数

## 约束与质量红线

- 总字数 ≥ 20000 字，使用 `wc -m` 精确验证
- 至少 8 个有效来源，其中 S/A 级来源不少于 3 个
- 每个核心概念必须配有可运行代码示例（禁止伪代码）
- 必须搜索微信公众号内容（`site:mp.weixin.qq.com`），不可跳过
- 教程开头必须标注更新日期和适用版本
- 若主题术语存在歧义，必须先向用户确认，不得臆断

## 示例用法

```
当您要求 Claude "写一份 LoRA 微调的完整教程"时：
1. 分析知识点：LoRA（Low-Rank Adaptation）→ 算法/模型类型，面向有一定深度学习基础的读者
2. 并行搜索：arXiv 论文、HuggingFace 文档、知乎/CSDN 博客、微信公众号深度文章
3. 从 15+ 来源中交叉验证关键概念，按 S/A/B/C 评级筛选
4. 选用算法/模型类大纲，增加数学基础和变体对比章节
5. 生成约 25000 字的完整教程，含 LoRA 原理、数学推导、代码实现、微调实战
6. 自检通过后保存为 LoRA_从入门到精通.md
```

## 要求

- WebSearch / WebFetch 工具权限（用于多源资料搜集）
- Agent 工具权限（用于并行搜索）
- Bash 工具权限（用于 `wc -m` 字数验证和 `date` 获取日期）
- 对当前工作目录的写入访问
