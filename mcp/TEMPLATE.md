# 工具文档模板

新增工具时，按以下五段结构编写，保持格式统一。

**核心原则**：
- **实事求是**：所有内容必须以官方文档为优先来源，其他权威网站为辅助。不可杜撰、不可推测
- **突出核心**：功能介绍、优劣对比中，最特色、最突出、最核心的功能排在最前面。让用户一眼看到该工具最擅长什么、什么场景下是最优解
- **来源可溯**：关键数据（Star 数、功能描述、配置方式）需来自官方 README 或文档

### 各平台配置

按 AI 工具分小节。根据协议类型选择配置方式：

- **MCP 工具**：使用下方 JSON 配置
- **Skill / CLI 工具**：使用 CLI 安装命令，无需 JSON 配置

配置参数约定：`command` 统一使用**数组格式** `["npx", "xxx"]`，避免字符串解析歧义。

#### Claude Code

**MCP 格式：**

```json
{
  "mcpServers": {
    "{工具名}": {
      "command": ["npx", "{包名}@"latest"],
      "args": ["--flag"],
      "enabled": true
    }
  }
}
```

验证：`claude mcp list` 检查 {工具名} 是否在列表中。

**Skill 格式：**

在对话中告诉 Claude：
```
Install {工具名}. Skill source: {GitHub 地址} . Verify it works after installation.
```

验证：在对话中尝试调用该工具的某个命令。

#### OpenCode

**MCP 格式：**

```json
{
  "mcp": {
    "{工具名}": {
      "command": ["npx", "{包名}@"latest"],
      "enabled": true,
      "type": "local"
    }
  }
}
```

验证：重启 OpenCode 后，在对话中询问 "有哪些可用的 MCP tools" 检查是否出现该工具。

**Skill 格式：**

在对话中告诉 OpenCode：
```
Install {工具名}. Skill source: {GitHub 地址} . Verify it works after installation.
```

#### Codex

```toml
[mcp_servers.{工具名}]
```



#### Cursor / Windsurf

### 卸载

| 工具类型 | 卸载方式 |
|---------|---------|
| MCP（JSON 配置） | 删除对应配置块即可 |
| Skill（CLI 安装） | 根据工具文档执行卸载命令 |
| npx 缓存包 | 自动管理，无需手动删除 |

### 更新

| 工具类型 | 更新方式 |
|---------|---------|
| MCP（npx） | 将版本标签改为 `@latest`，npx 会自动使用最新版 |
| MCP（全局安装） | `npm update -g {包名}` 或 `pip install --upgrade {包名}` |
| Skill | 重新执行安装命令，覆盖旧版本 |
