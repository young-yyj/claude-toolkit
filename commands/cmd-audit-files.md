# cmd-audit-files

当用户执行 `/cmd-audit-files` 时，按以下流程操作。

## 执行流程

### 1. 定位上一轮操作

从当前对话历史中提取最近一次用户请求及其对应工具调用记录。如果上一轮没有文件操作，直接回复"上一轮对话无文件操作记录"。

### 2. 提取文件操作

扫描上一轮中的所有工具调用，筛选出涉及文件变更的操作：

- **Write** — 创建或覆盖文件
- **Edit** — 修改文件
- **Bash** — 仅提取包含 `mv`、`cp`、`rm`、`rename`、`git mv` 等文件操作的命令
- **NotebookEdit** — Notebook 编辑

### 3. 分类规则

按以下类型归类每条操作：

| 操作类型 | 判断依据 |
|---------|---------|
| 创建文件 | Write 工具（目标路径之前不存在） |
| 修改文件 | Edit 工具 |
| 删除文件 | Bash 中的 `rm`、`git rm` |
| 移动/重命名 | Bash 中的 `mv`、`rename`、`git mv` |
| 覆盖文件 | Write 工具（目标路径之前已存在） |

无法确定类型的归入"其他"。

### 4. 输出格式

以表格形式输出，默认包含以下列：

| 操作类型 | 文件路径 | 操作详情 |

每行的"操作详情"写一句简洁说明：创建内容/修改了什么/从何处移动。

表格后附加统计汇总：

```
共计 N 项操作：创建 X 项 | 修改 Y 项 | 删除 Z 项 | 移动/重命名 W 项
```

仅出现非零项。

### 5. 范围说明

若用户追加时间范围（如 `/cmd-audit-files 最近3轮`），按指定范围提取而非仅上一轮。
